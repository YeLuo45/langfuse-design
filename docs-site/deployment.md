# 部署

Langfuse 支持多种部署方式，从本地开发到生产环境全覆盖。

## 1. Docker Compose（本地/开发）

适用于本地开发或小型部署场景。

```bash
docker-compose up
```

环境变量配置请参考 `.env.dev.example` 文件。

## 2. Kubernetes（生产）

适用于生产环境部署。

配置文件请参考 `.env.prod.example` 文件。

## 3. Self-Hosted

自托管部署支持 Docker 和 Kubernetes 两种方式。

详细文档请参阅：[langfuse.com/docs/deployment/self-host](https://langfuse.com/docs/deployment/self-host)

## 4. Langfuse Cloud

托管服务，无需自行运维，开箱即用。

访问地址：[cloud.langfuse.com](https://cloud.langfuse.com)

## 部署架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Langfuse 部署架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐      │
│  │  Web 容器    │    │  Worker 容器 │    │    Redis     │      │
│  │ (Next.js)   │    │ (后台任务)   │    │ (队列/缓存)  │      │
│  └─────────────┘    └─────────────┘    └─────────────┘      │
│         │                  │                                  │
│         └──────────────────┼──────────────────────────────   │
│                            │                                  │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐      │
│  │   Postgres  │    │  ClickHouse  │    │     S3      │      │
│  │ (事务数据)   │    │  (分析数据)   │    │ (文件存储)   │      │
│  └─────────────┘    └─────────────┘    └─────────────┘      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 组件说明

| 组件 | 说明 |
|------|------|
| **Web 容器** | Next.js 应用，处理前端页面和 API 请求 |
| **Worker 容器** | 后台任务处理器，负责异步任务执行 |
| **Postgres** | 事务性数据库，存储配置、用户、追踪等核心数据 |
| **ClickHouse** | 分析型数据库，存储海量事件数据用于分析查询 |
| **Redis** | 消息队列和缓存层，提升系统性能 |
| **S3** | 对象存储服务，存储文件、附件等大型二进制数据 |
